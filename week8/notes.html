Chapter 8: CSS3 Transforms and Transitions

Transforms

The CSS3 transform property lets you translate, rotate, scale, and/or skew any element on the page.

Translation

Translation functions allow you to move elements left, right, up, or down. These functions are similar to the behaviour of posistion: relative; when declaring top and left, moving elements up and down or left and right along the x and y axes.

A translated element can only be moved relative to its current position.

Scaling

The scale(x,y) function scales an element by the defined factors horizontally then vertically.

As with translate, you can also use the scaleX(x) or scaleY(y) functions. These functions will scale only the horizontal dimensions or only the vertical dimensions respectively.

Rotation

The rotate() function rotates an element around the point of orgin by a specified angle value.

Changing the Origin of the Transform

The orgin of transform can be controled by using the transform-origin property.

It has the same syntax as the background-position

Transitions

Transitions allow the values of CSS properties to change over time, essentially providing simple animations.

The transition-property property defines the CSS properties of the element that should be transitioned, with all for all properties being the default.

The transition-duration Property

The transition-duration property sets how long the transition will take: the duration of time it takes to go from the defualt state to the transitioned state.

The transition-timing-function Property

The transition-timing-function lets you control the pace of the transition in even more granular detail.

You can describe your timing function more precisely by defining your own cubic-bezier function.

The transition-delay Property

The transition-delay property make it possible to add a delay before the transition begins.

Negative Delays

A negative time delay that's less than the duration of the entire transition will cause it to start immediately, but it will start partway through the animation.

Animations

CSS animations, unlike transitions, allow you to control each step of an animation via keyframes.

Chapter 12:Canvas, SVG, and Drag and Drop

Creating a canvas Element

Setting width and height

The canvas element takes both a width and height attribute, which must also be set.

Setting the width and height attributes determine how large the canva's coordinate system is. If you don't specify width and height, the canvas element will defualt to a width of 300 and a height of 150. If you set the wedth and height for a canvas only in CSS, the canvas element will be 300 by 150, and the CSS properties will simply determine how large the box is that displays the image.

Drawing Other Shapes by Creating Paths

Canvas is not limited to drawing rectangles. Unlike rectangles and squares, however, there's no built-in method for drawing circles or other shapes. To draw more interesting shapes, you must first lay out the path of the shape.

Paths create a blueprint for your lines, arcs, and shapes, but paths are invisible until you give them a stroke.

Saving Cavas Drawings

If you create an image programmatically using the Canvas API, but decide you'd like to have a local copy of the drawing, you can use the API's toDataURL method to save our drawing. toDataURL creates a URL with the image in it, (either a png or jpg, whichever is specified).

Manipulating Images

Once an image is drawn on the canvas, you can use the getImageData method from the Canvas API to manipulate the pixels of that image.

Canvas versus SVG

Canvas allows for pixel manipulation. One downside of canvas is that it operates in what’s known as immediate mode. This means that if you ever want to add more to the canvas, you’re unable to simply add to what’s already there. Every time you finish drawing a shape, the canvas no longer has access to that shape, because it won’t persist as an object that you can modify. So if you want to add to what you’ve already drawn on the canvas, you must redraw the new shape from scratch.

what you draw to SVG is accessible via the DOM, because its mode is retained mode, meaning that the structure of the image is preserved in the XML document that describes it. SVG also has, at this time, a more complete set of tools to help you work with it, such as the Raphaël library and Inkscape. However, since SVG is a file format—rather than a set of methods that allows you to dynamically draw on a surface—you can’t manipulate SVG images the way you can manipulate pixels on canvas.
